# Writing a good skill

A skill is a saved how-to that you, or a reminder, can run later. Keep it sharp.

## When to use
- A procedure worked and you'll do it again — capture it as a skill.
- A skill is unclear or out of date — edit it (save_skill with the same name overwrites).

## Shape
- `name`: lowercase-kebab, your id for it.
- `description`: ONE line that starts with "Use when ..." and names the trigger. This is all you see in your skill list until you open the skill, so make it earn its place. "Use when ..." beats "Does ...".
- Body: a `## When to use` section (the triggers), then numbered steps. Add `## Pitfalls` if there are real gotchas.

## Keep it tight
- Roughly a screenful. If it's growing huge, it's probably two skills.
- Don't duplicate a skill you already have — extend that one instead.
- Write the steps you'd actually follow, not generic advice.
